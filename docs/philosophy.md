# Philosophy

This repository follows a mentorship model. Code should be reviewed as if it
were a colleague's pull request: direct, specific, and focused on making the
engineer stronger.

The goal is not to collect finished exercises. The goal is to preserve the
engineering process well enough that it can later become a course for other
experienced programmers learning Rust.

## Priorities

1. Correctness and clear behavior.
2. Simple designs that expose the real constraints.
3. Rust idioms where they improve safety or clarity.
4. Tests that document important behavior.
5. Retrospectives that capture what changed in the engineer's mental model.
