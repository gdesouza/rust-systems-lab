# Course Notes

This file collects durable lessons that should survive beyond any single
exercise. It should stay high-level; detailed examples belong in the relevant
exercise, project, or topic note.

## Emerging Themes

- Rust systems work rewards explicit ownership of I/O, allocation, and error
  handling decisions.
- A working program is only the first checkpoint. The useful learning often
  appears during review and revision.
- Course material should be grounded in real mistakes, not polished examples
  invented after the fact.

## Go vs Rust Thread

Each exercise should include a short reflection aimed at an experienced Go
engineer learning Rust:

- What felt similar?
- What required a different mental model?
- Which Rust constraints improved the design?
- Where did Rust add ceremony, and was it worth it?
